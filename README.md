# Интернет-каталог магазина спортивного питания

Веб-приложение — интернет-каталог магазина спортивного питания, разработанное в рамках выпускной квалификационной работы.  
Проект реализован на Django по архитектуре MVT и ориентирован на серверный рендеринг HTML без использования SPA-подхода.

## Функциональность

### Общедоступные страницы
- Главная страница (каталог товаров)
- Страница «О нас»
- Страница «Контакты»
- Поиск товаров

### Каталог
- Фильтрация по бренду, категории и цене
- Сортировка (по умолчанию, сначала дороже, сначала дешевле)
- Пагинация

### Авторизованные пользователи
- Корзина (как временное состояние заказа)
- Список желаемого
- Оформление заказа:
  - выбор офлайн-магазина
  - добавление комментария к заказу
- Профиль пользователя:
  - редактирование личных данных
  - просмотр истории заказов
  - краткий список товаров из списка желаемого
  - выход из аккаунта

## Особенности реализации
- Корзина не представлена отдельной сущностью в базе данных и реализована как временное состояние сущности «Заказ»
- Используется кастомная модель пользователя, созданная на основе стандартной модели Django и расширенная под задачи проекта
- Серверный рендеринг HTML (без использования React/Vue)
- Классическая MVT-архитектура Django

## Установка и запуск

### Требования
- Python 3.x
- MySQL

Перед запуском необходимо настроить параметры подключения к базе данных в файле `settings.py`.

1. Клонировать репозиторий:
```bash
git clone https://github.com/ks17exs/ns
```

2. Создать виртуальное окружение и установить зависимости:
```bash
python -m venv venv
source venv/bin/activate  # Linux / macOS
venv\Scripts\activate     # Windows
pip install -r requirements.txt
```

3. Настроить подключение к базе данных и SECRET_KEY в файле `settings.py`.

Применить миграции:
```bash
python manage.py migrate
```

4. Запустить сервер разработки:
```bash
python manage.py runserver
```

### Примечание
Файл `settings.py` содержит заглушки для конфиденциальных данных.
Для запуска проекта необходимо указать реальные параметры подключения к базе данных и `SECRET_KEY`.
